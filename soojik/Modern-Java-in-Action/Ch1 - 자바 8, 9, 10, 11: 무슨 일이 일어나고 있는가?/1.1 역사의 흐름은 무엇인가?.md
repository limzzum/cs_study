# 1.1 역사의 흐름은 무엇인가?  

### Java 8
자바 역사를 통틀어 가장 획기적이고 큰 변화가 일어난 버전

__8 이전__
```java
Collections.sort(box, new Comparator<Apple>() {
    public int compare(Apple a1, Apple a2) {
            return a1.getWeight().compareTo(a2.getWeight());
    }
}
```
이랬던 코드가  

__8 이후__
```java
box.sort(comparing(Apple::getWeight));
```
과 같이 람다식 사용 등으로 자연어에 가까워 이해하기 쉽게, 간결해졌다.  

**멀티코어 CPU 대중화**와 같은 하드웨어적인 변화도 Java 8 버전에 영향을 주었다.  
이전까지의 자바 프로그램은 하나의 코어만을 사용하였고, 나머지 코어를 활용하려면 스레드를 사용하는 것이 좋다고 여겨졌다.  
스레드를 사용하기 어려운 상황 속, 자바는  
`자바 1.0` - 스레드, 락, 메모리 모델 지원  
`자바 5` - 스레드 풀, 병렬 실행 컬렉션  
`자바 7` - 포크/조인 프레임워크를 지원(했지만 개발자가 활용하기 쉽지 않았음)  

각 버전에서 병렬 실행 환경을 쉽게 관리하고 에러가 덜 발생하는 방향으로 개선되어나갔다.  

`자바 8`은
- 간결한 코드
- 멀티코어 프로세서의 쉬운 활용  

을 기반으로 하며,  

- 스트림 API(데이터 베이스 질의 언어에서 표현식을 처리하는 것처럼 **병렬 연산**을 지원하는 API)  
데이터베이스 질의 언어에서 <span style='color: orange'>고수준 언어로 원하는 동작</span>을 표현 -> 구현에서 최적의 저수준 실행 방법을 선택  
따라서 스트림을 이용하면 에러를 자주 일으키며, 멀티코어 CPU를 이용하는 것보다 비용이 훨씬 비싼 `synchronized`를 사용하지 않아도 된다.
- 메서드에 코드를 전달하는 기법
- 인터페이스의 디폴트 메서드  

와 같은 새로운 기술을 제공한다.
