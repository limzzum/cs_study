## 인스턴스화를 막으려거든 private 생성자를 사용하라

```
정적 메서드 혹은 정적 멤버만 담은 클래스를 만들 수 있다.
최근 만들었던 것 중 예를 들면 정적 멤버만 담은 유틸리티 클래스로,
API 응답 메시지들을 모아 놓는 클래스를 만들었었다.

그런데 나는 이것을 인스턴스로 만들어 쓰려고 한 것이 아니다.
하지만 생성자를 명시하지 않으면 자동으로 기본 생성자가 생성된다.
이러한 문제가 실제로 API들에도 나타나 의도치 않게 인스턴스화 할 수 있게 된 클래스가 목격된다.

이러한 문제를 막기 위해 private 생성자를 추가하여 해결할 수 있다.
 
```
